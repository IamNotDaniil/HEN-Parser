# HEN-Parser

HEN-Parser — сервис для сбора вакансий из внешних источников и аналитики рынка: востребованные технологии, зарплатные вилки, распределение по регионам и динамика спроса по стеку.

## Текущий статус

### Уже сделано

- Подготовлен стартовый Python-парсер для `hh.ru`.
- Добавлен CLI для выгрузки вакансий по поисковому запросу.
- Реализовано сохранение сырых данных в `JSON` и `CSV`.
- Подготовлена стартовая схема PostgreSQL для хранения вакансий, компаний, локаций и снимков зарплат.
- Добавлен `docker-compose.yml` для локального запуска PostgreSQL.

## Архитектура первого этапа

На текущем этапе проект состоит из трёх основных частей:

1. **Parser** — Python-клиент к `hh.ru`, который получает вакансии по ключевому слову.
2. **Raw storage** — файловое сохранение выгрузок в `data/raw` для отладки и повторной загрузки.
3. **Database** — PostgreSQL со стартовой схемой для дальнейшего ETL и аналитики.

## Структура репозитория


## Быстрый старт

### 1. Подготовить окружение

```bash
./scripts/bootstrap.sh
```

### 2. Запустить PostgreSQL

```bash
docker compose up -d
```

После старта контейнера схема из `db/schema.sql` будет применена автоматически.

### 3. Выгрузить вакансии с hh.ru

```bash
python3 -m parser.main "Python" --pages 2 --per-page 20
```

Пример с указанием региона:

```bash
python3 -m parser.main "Go developer" --area 1 --pages 1 --per-page 10
```

По умолчанию результаты сохраняются в `data/raw/`.

## Что именно делает парсер

CLI-команда:

- отправляет запрос в `hh.ru API`;
- забирает несколько страниц вакансий;
- извлекает ключевые поля вакансии;
- сохраняет полный сырой payload для последующей обработки;
- создаёт два файла:
  - `JSON` — для повторной загрузки и отладки;
  - `CSV` — для быстрого просмотра и первичного анализа.

## Схема данных PostgreSQL

В `db/schema.sql` уже подготовлены базовые таблицы:

- `source_runs` — история запусков сбора;
- `companies` — компании;
- `locations` — города / локации;
- `vacancies` — основная таблица вакансий;
- `salary_snapshots` — история зарплатных значений.

Эта схема — база для следующего шага: загрузки raw-данных в БД и построения аналитических представлений.
